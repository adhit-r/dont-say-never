Why This Study Exists

In CodeCoach, scanner findings were converted into persistent coding-agent rules. One pilot case looked paradoxical: a prohibition against an unsafe pattern made one model produce that pattern more often.

Pilot signal

A narrow “do not do X” failure motivated the larger study.

Replication question

Does rule wording generalize across models and CWEs?

Experiment

2,160valid model trials
6frontier coding models
6security prompts
3conditions per prompt
  • Control: no rule injected.
  • Negative: prohibition-framed rule.
  • Positive: safe alternative-framed rule.

Main Result

Rules reduce vulnerable code on every model. The reliable effect is the presence of a targeted rule. The polarity of that rule is not consistently predictive.

Interpretation

The pilot was real but narrow.

A single paradoxical case motivated the study; the larger replication shows it is not the general rule.

Persistent rules are useful guardrails.

Across GPT and Claude stacks, targeted security rules lowered vulnerable outputs.

Wording is model- and prompt-dependent.

Teams should evaluate rule files against their own prompts, not rely on a universal phrasing heuristic.

For Practitioners

  • Keep rule files short, targeted, and security-reviewed.
  • Prefer measurable guardrail tests over prompt folklore.
  • Treat AGENTS.md and CLAUDE.md as security-critical configuration.
Security scope. Prompts cover CWE-style insecure code: dynamic execution, weak hashing, cleartext HTTP URLs, and insecure randomness.
Ethics. No human subjects were studied. The dataset contains synthetic prompts and generated code outputs.
